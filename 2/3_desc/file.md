## описание предметной области

Сущности: Студенты, Преподаватели, Курсы, Модули, Уроки (видео, текст, тест), Домашние задания, Проверка ДЗ (с системой рецензий), Прогресс студента. 

Сложность: 
- Сложная структура курса (курс → модули → уроки). 
- Система проверки ДЗ: студент сдал → преподаватель проверил → оставил комментарии → студент исправил → новая проверка. 
- Хранение прогресса по каждому уроку для каждого студента. 

## описание проблем

1. Классический SQL:
    - развертывание базы данных
    - подключение к базе данных из python

2. BaaS:
    - создание таблиц в интерфейсе, а не выполнением кода/импортом файла
    - заполнение таблиц данными в интерфейсе, а не выполнением кода/импортом файла

## итоговый вывод

1. Классический SQL предпочтительно использовать для:
    - Проекты с большим объёмом данных и сложными зависимостями (например, банковские системы, ERP, CRM).
    - Сложные транзакционные системы, где важна консистентность данных (ACID).
    - Проекты с требованием высокой производительности и оптимизации запросов.
    - Системы с кастомной бизнес-логикой, где нужно точно контролировать структуру БД и индексы.
    - Долгосрочные корпоративные проекты, где важно масштабирование и гибкость архитектуры.

2. BaaS предпочтительно использовать для:
    - Мобильные и веб-приложения MVP (Minimum Viable Product), когда нужно быстро протестировать идею.
    - Проекты с небольшой командой или без dedicated backend-разработчика.
    - Приложения с динамическими структурами данных, где не всегда нужна строгая схема.
    - Проекты, требующие встроенной аутентификации, пуш-уведомлений и облачного хранения файлов.
    - Стартапы, где скорость разработки важнее глубокой оптимизации.
